# Step 5: 문서 작성과 핸드오프 (write-docs)

## 목적

지금까지 확정된 내용을 문서 3종으로 굳히고, 다음 단계로 넘긴다.
새로운 내용을 만들지 않는다 — 이 step은 기록이지 창작이 아니다.
확정 안 된 내용이 문서에 필요하면 이전 step으로 돌아가 물어본다.

## INSTRUCTIONS

### 1. 문서 3종 작성

`./templates/` 의 템플릿을 사용해 작성한다:

| 파일 | 템플릿 | 내용 |
|---|---|---|
| `{output_dir}/00-brief.md` | `templates/00-brief.md` | 문제·타겟·핵심 가치·도메인 역학·성공 기준 |
| `{output_dir}/01-product.md` | `templates/01-product.md` | 사용자 여정·화면 목록·플로우 (**UI 디자인 입력용**) |
| `{output_dir}/02-roadmap.md` | `templates/02-roadmap.md` | 개발 단계 + 도메인 근거 + 비개발 작업 |

각 문서는 1~2페이지를 넘기지 않는다. 길어지면 구체화가 아니라 비대화다 —
빼도 다음 단계가 곤란하지 않은 내용은 뺀다.

### 2. 승인

세 문서의 초안을 보여주고 `[A]` 승인을 받는다. **승인 전에는 파일로 저장하지
않는다.** 수정 요청은 반영 후 다시 보여준다.

HALT하고 사용자 승인을 기다린다.

### 3. 저장과 핸드오프

승인되면 `{project_root}/{output_dir}/`에 저장하고, 다음 단계를 안내한다:

---

기획 문서 3종을 `{output_dir}/`에 저장했습니다.

**다음 단계:**
- **UI 디자인** — `01-product.md`를 디자인 툴/스킬의 입력으로 사용하세요.
  화면 목록·상태·플로우가 와이어프레임의 명세입니다.
- **기술 설계·개발** — Phase 1 범위로 design-doc 스킬을 실행하면
  `00-brief.md`와 `02-roadmap.md`가 배경 자료가 됩니다.

---

**END OF WORKFLOW.**
