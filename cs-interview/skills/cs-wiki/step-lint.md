# Step Lint: 위키 점검

위키 건강검진. 문제를 찾아 보고하고, 수정은 사용자 승인 후 진행한다.

## RULES

- 발견한 문제를 **자동 수정하지 않는다.** 먼저 보고하고 사용자 승인을 받는다.
- 점검은 읽기 작업이다. `{wiki_root}`를 수정하는 건 승인 이후뿐이다.


## INSTRUCTIONS

### 1. 점검 수행

루트 `index.md`와 `log.md`로 전체 윤곽을 잡고, 페이지들을 훑어 다음을 찾는다:

- **모순** — 페이지 간 설명이 충돌하는 곳.
- **낡은 서술** — 새 정보로 갱신이 필요한 곳.
- **고아 페이지** — 인바운드 링크가 없는 페이지.
- **누락 개념** — 본문에서 언급됐지만 자체 페이지가 없는 중요 개념 (생성 후보).
- **끊긴 구조** — 누락된 교차링크, 비어 있는 핵심 섹션, index에 빠진 페이지.

### 2. 보고

발견 항목을 분류해 목록으로 제시한다. 각 항목에 위치(concept ID)와 한 줄 설명, 권장 조치를 붙인다.

> **점검 결과** ({N}건)
>
> **모순**
> - `network/tcp-vs-udp` ↔ `network/udp`: {무엇이 충돌하는지} → 권장: {조치}
>
> **누락 개념 (생성 후보)**
> - `network/quic` — tcp-vs-udp에서 언급되나 페이지 없음.
>
> ... (이하 동일)

### 3. 수정 (승인 후)

사용자가 고칠 항목을 고르면 처리한다:

- 페이지 작성/갱신이 필요하면 `./step-ingest.md`의 절차를 따른다.
- 처리한 내용은 `log.md`에 `* **점검**: ...`으로 기록한다.

작업 완료. 추가 요청이 있으면 `./step-01-route.md`로 다시 분류한다.
