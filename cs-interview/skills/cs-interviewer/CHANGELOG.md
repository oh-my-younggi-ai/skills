# cs-interviewer — Changelog

## iter-01 · 2026-06-18 — initial draft

- 스킬 첫 버전. step-file 아키텍처(setup → interview → evaluate-record)
- `cs-wiki`를 정답 기준으로 읽고(**읽기 전용**), 약점 가중 + 전체 랜덤 혼합 출제
- 답변에서 키워드 추출 → 꼬리 질문(깊이 파기 / 함정 찌르기 / 빈 곳 메우기, 개념당 최대 2개)
- 면접 기록을 `cs-interview-records/`에 **분리** 저장(sessions/·log.md·mastery.md) — 지식 위키 오염 방지
- 위키에 없는 개념은 "정답 기준 미검증"으로 표기 + `cs-wiki` 정리 제안(직접 안 씀)
- **검증:** JPA 5개념 실전 모의면접으로 직접 검증 — 키워드 꼬리질문·위키 기준 평가·기록 분리·약점 추적 동작 확인

### Fixed
- **답변 대필 버그**: 질문 직후 모델이 턴을 끝내지 않고 사용자 답변을 추측·대필하며 다음으로 넘어가는
  문제. workflow INVARIANT와 step-02에 "질문 직후 즉시 턴 종료 / 사용자 답변 대필·추측 금지 /
  실제 입력이 온 뒤에만 진행"을 명시.

### Changed (보고서 개선)
- **대화 원문 기록**: 세션 보고서가 답변을 요약하지 않고 면접관 질문·사용자 답변을 **원문 그대로** 기록.
- **점수 추가**: 채점 rubric(정확성·완전성·깊이)으로 개념별 0~100점 + 종합 점수. mastery에 점수 컬럼.
- **인라인 첨삭**: 틀리거나 미흡한 답변 **바로 아래**에 `✍️ 첨삭`으로 교정.
- **공부 추천**: 보고서 끝에 약점/심화 주제를 한 줄 이유·링크와 함께 추천.
