# Step 2: 질문을 통한 사전 정보 수집

## RULES

- 한 라운드에 4~6개의 질문만 제시한다.
- 이미 답변이 충분한 카테고리는 다시 묻지 않는다.
- 사용자는 일부 질문에 "모르겠음" / "나중에 결정"으로 답할 수 있다 — 그대로 `{qa_log}`에 기록하고 넘어간다.
- **EARLY EXIT** = 이 step을 즉시 중단하고 지정된 파일로 이동한다.


## INSTRUCTIONS

### 1. 라운드 시작

`{question_round}` += 1


### 2. 질문 생성

`{topic}`과 지금까지의 `{qa_log}`를 바탕으로, 아직 다뤄지지 않았거나 불명확한 카테고리에 대한 질문을 만든다:

- **배경/목적** — 왜 필요한가, 어떤 문제를 해결하는가
- **범위** — 무엇을 포함/제외하는가
- **사용자/이해관계자** — 누가 사용하는가, 누구에게 영향을 주는가
- **기존 시스템과의 관계** — 영향받는 모듈/파일, 연동 지점, 의존성
- **제약사항** — 기술 스택, 성능, 일정, 호환성
- **완료 기준** — 무엇이 되면 끝났다고 볼 수 있는가, 검증 방법

이미 충분히 답변된 카테고리는 건너뛴다. 카테고리당 1개 내외, 총 4~6개로 구성한다.


### 3. 질문 제시

HALT하고 번호를 매겨 제시한다:

---

**라운드 {question_round} — 추가로 확인하고 싶은 내용**

1. {질문 1}
2. {질문 2}
3. {질문 3}
4. {질문 4}

---

> 답변하기 어려운 항목은 "모르겠음" 또는 "나중에 결정"이라고 적어주셔도 됩니다.

HALT하고 사용자 입력을 기다린다.


### 4. 답변 기록

사용자 답변을 카테고리별로 `{qa_log}`에 추가한다.


### 5. 충분성 판단

- **모든 핵심 카테고리가 다뤄졌다면** → 단계 6으로 진행.
- **빠진 카테고리가 있고** `{question_round}` < `{max_question_rounds}` **이면** → 단계 1로 돌아가 다음 라운드 진행.
- **빠진 카테고리가 있고** `{question_round}` >= `{max_question_rounds}` **이면** HALT:

  > 다음 항목은 아직 명확하지 않습니다: {빠진 카테고리 목록}
  >
  > `[A]` 이대로 설계 문서 작성 — 불명확한 부분은 "미정"으로 표시합니다.
  > `[Q]` 질문 계속하기 — 한 라운드 더 진행합니다.

  - **[A]**: 단계 6으로 진행.
  - **[Q]**: 단계 1로 돌아가 다음 라운드 진행 (`{max_question_rounds}` 제한 무시).


### 6. 작성 준비 완료 확인

`{question_round}` > 1 이었던 경우에만 HALT:

> 설계 문서를 작성하기에 충분한 정보가 모인 것 같습니다.
>
> `[A]` 설계 문서 작성 시작
> `[Q]` 질문 더 받기 — 추가로 다루고 싶은 부분이 있나요?

- **[A]**: 단계 7로 진행.
- **[Q]**: 단계 1로 돌아가 다음 라운드 진행.

`{question_round}` == 1 이었던 경우 (즉시 모든 카테고리가 충족된 경우) 바로 단계 7로 진행한다.
