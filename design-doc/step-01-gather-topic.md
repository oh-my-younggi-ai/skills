# Step 1: 주제 파악 및 저장 경로 결정

## RULES

- 주제는 항상 사용자에게 직접 물어서 결정한다. 예외 없음.
- 이전 대화에서 주제가 언급되었더라도 반드시 다시 확인한다.
- **EARLY EXIT** = 이 step을 즉시 중단하고 지정된 파일로 이동한다.


## INSTRUCTIONS

### 1. 주제 파악

HALT하고 사용자에게 묻는다:

> 무엇을 설계하고 싶으신가요? 한 줄로 설명해 주세요.
> (예: "알림 기능에 이메일 채널 추가", "결제 모듈 리팩토링 설계")

`{topic}` = 사용자 답변


### 2. 슬러그 및 저장 경로 제안

`{topic}`을 바탕으로 영문 kebab-case 슬러그를 제안한다 (예: `email-notification-channel`).

`{slug}` = 제안한 슬러그
`{design_doc_path}` = `{project_root}/{output_dir}/{slug}.md`

HALT하고 제시한다:

> 설계 문서는 다음 경로에 저장할 예정입니다:
> `{design_doc_path}`
>
> `[A]` 이대로 진행
> `[E]` 경로/파일명 변경 — 원하는 경로를 알려주세요

- **[A]**: 단계 3으로 진행.
- **[E]**: 사용자가 알려준 경로로 `{design_doc_path}`를 갱신하고 단계 3으로 진행.


### 3. 기존 파일 확인

`{design_doc_path}`이 이미 존재하면 HALT:

> 해당 경로에 이미 파일이 있습니다 (`{design_doc_path}`).
>
> `[O]` 덮어쓰기 — 새로 작성합니다
> `[E]` 다른 경로 사용 — 원하는 경로를 알려주세요
> `[X]` 중단

- **[O]**: 단계 4로 진행 (기존 내용은 무시하고 새로 작성).
- **[E]**: 새 경로로 `{design_doc_path}`를 갱신하고 단계 3을 다시 확인한다.
- **[X]**: HALT.

존재하지 않으면 단계 4로 진행한다.


### 4. 초기화

`{qa_log}` = 빈 상태로 초기화
`{question_round}` = 0
